Media Query vs. Container Query

一個用於響應**整個頁面**的佈局，另一個則用於響應**元件本身**的空間。讓我們來看看它們的區別。

Media Queries (媒體查詢)

Media Query 讓你的內容可以響應**瀏覽器視窗 (viewport) 的大小**。這是傳統響應式設計的基石。你可以把它想像成：「如果螢幕寬度超過 768px，就把側邊欄顯示出來」。它的判斷基準永遠是整個頁面。

Container Queries (容器查詢)

Container Query 讓元件可以響應其**父層容器 (parent container) 的大小**，而不管整個視窗有多大。這讓你可以建立真正模組化、可重複使用的元件。你可以把它想像成：「如果我所在的容器寬度超過 350px，就把我的排版從垂直變為水平」。

👇 同一個元件，在不同寬度的容器中，表現不同 👇

總結比較

特性 Media Query Container Query
響應對象 整個瀏覽器視窗 (Viewport) 父層容器 (Parent Container)
使用場景 頁面級別的宏觀佈局 (如導航欄、側邊欄、頁面網格) 元件級別的微觀佈局 (如卡片、個人資料小工具)
核心概念 頁面應該如何適應不同的螢幕尺寸 元件應該如何適應它被放置的任何空間
CSS 語法 @media (min-width: 768px) { ... } @container (min-width: 350px) { ... }
